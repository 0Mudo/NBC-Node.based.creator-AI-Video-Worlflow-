## 二、新功能详情

### 1. 云端素材库（OSS Browser）

在左侧面板新增"云端"标签页，直接连接到阿里云 OSS bucket。用户可以：

- 浏览 OSS bucket 的完整目录树（按 prefix 层级展开）
- 在网格视图和列表视图之间切换
- 搜索当前目录下的文件
- **直接拖拽 OSS 素材到画布**，供 Seedance 2.0 等节点使用（自动携带签名 URL）

**影响范围**：electron/main.ts（新增 4 个 IPC）、electron/preload.ts、electron.d.ts、新组件 OssBrowser.tsx、MainLayout.tsx

### 2. 云端上传与一键同步

- **单文件上传**：点击"上传"按钮，从本地选择图片/视频文件，上传到当前 OSS 目录，带实时进度条。
- **一键同步本地到 OSS**：点击"同步本地素材"，自动扫描本地素材库目录（如 `H:\素材库`），将尚未在 OSS 上的文件批量上传，去重跳过已存在文件。

### 3. 新建空白三卡

灵感编辑器（角色卡/场景卡/物品卡）新增"新建空白"按钮。点击后创建一个全新的空卡片，可自由编辑文本（自由文本/结构化两种模式）并绑定图片。

### 4. 素材库视频首帧预览

视频素材现在自动使用 canvas 提取首帧作为缩略图显示，替代原来直接渲染 `<video>` 元素的方式。缩略图缓存在内存中，重复显示无需重新提取。

**影响范围**：新组件 VideoThumbnail.tsx、AssetCard.tsx

### 5. 素材库删除功能增强

删除素材时弹出确认对话框，提供两个选项：

- **仅从 NBC 移除**：从素材库列表中移除，本地文件保留
- **放入回收站**：从 NBC 移除的同时，将本地文件移入系统回收站（通过 Electron `shell.trashItem` API）

OSS 素材的删除也从云端删除。

**影响范围**：新组件 DeleteConfirmDialog.tsx、AssetBrowser.tsx、AssetCard.tsx、electron/main.ts

### 6. 节点连接判定区域增大

节点端点（Handle）的 CSS 伪元素从原来的 9×9px 扩大至 28×28px 的透明点击区域，视觉外观不变。连接节点时不再需要精确对准小圆点。

**影响范围**：index.css

### 7. 灵感编辑器图片绑定修复

角色卡右侧"角色形象编辑"区域的"从素材库绑定"按钮以前无响应——`handleImageBound` 函数虽然定义了但从未连接到任何 UI 弹窗。现在弹出完整的素材图片选择器（AssetImagePicker），显示所有图片/视频素材供挑选绑定，同时也支持更换绑定。

**影响范围**：新组件 AssetImagePicker.tsx、InspirationEditor.tsx

### 8. 参考图 base64 传输支持

GPT Image 和 Banana 节点的本地参考图现在自动转为 base64 data URI 传入 GRS API（API 文档明确支持 base64），解决了本地素材无法作为参考图的问题。Seedance 2.0 保持 URL 方式（Ark API 仅支持 URL）。

**影响范围**：promptResolver.ts（新增 `convertRefToDataUri` 函数）、useExecutionEngine.ts

## 三、Bug 修复

| Bug | 根因 | 修复方式 |
|-----|------|---------|
| 参考图不生效 | `assetInput` 节点的 `assetId` 为 `nbc://` 协议格式，被 HTTP-only 过滤器静默丢弃 | 新增 `resolveImageRefs` 解析 + `convertRefToDataUri` 转 base64 |
| 素材库删除偶发不生效 | 删除操作直接调 `removeAsset` 无确认，可能存在状态同步问题 | 重构为确认对话框流程，统一入口 |

## 四、技术债务/已知限制

- **纯本地文件作为 Seedance 参考图**：仍需先上传到 OSS 或已有 HTTP URL。可考虑后续增加自动 OSS 上传通道。
- **视频首帧提取依赖浏览器 API**：在非 Electron 环境下可能受 CORS 限制。
- **一键同步大文件**：当前为串行上传，后续可改为并发控制。
