# NBC 更新报告 — 2026-05-16

---

## 一、概述

本次更新涉及 3 项核心功能变更 + 1 项文档增强，聚焦于提升工作流的**结构化连接能力**和**用户交互体验**。

---

## 二、功能变更详情

### 变更 1：剧本/分镜节点拆分（原 ScriptNode → ScriptNode + StoryboardNode）

**变更前**：只有一个"剧本/分镜"节点（`script`），仅提供自由文本输入，与灵感编辑器的结构化数据无关联。

**变更后**：

| 节点类型 | 标识 | 颜色 | 图标 |
|---|---|---|---|
| 剧本节点（`script`） | 原节点更名 | 青色 #00cec9 | ScrollText |
| 分镜节点（`storyboard`） | 全新节点 | 橙红 #e17055 | Clapperboard |

**新增能力**：

- **剧本节点**：新增「灵感编辑器场次」下拉框，自动读取灵感编辑器第1步已保存的结构化场次（`ScriptScene[]`），选择后自动填充场次号、标题、动作描述
- **分镜节点**：新增「灵感编辑器分镜」下拉框，自动读取灵感编辑器第2步已保存的结构化镜头（`StoryboardShot[]`），选择后自动填充镜头号、景别、描述、对白、关联角色ID/场景ID/物品ID
- 分镜节点携带的 `characterIds` / `sceneId` / `itemIds` 可供时间线分析使用

**类型定义新增**：

```typescript
// flow.ts NodeType 新增 'storyboard'
// NodeData 新增字段：
scriptSceneId, scriptSceneNumber, scriptSceneHeading
storyboardShotId, storyboardShotNumber, storyboardShotDescription,
storyboardShotType, storyboardDialogue,
storyboardCharacterIds, storyboardSceneId, storyboardItemIds
```

**受影响文件**：
- `src/types/flow.ts` — 类型扩展
- `src/nodes/ScriptNode.tsx` — UI 更新为"剧本"
- `src/nodes/StoryboardNode.tsx` — 新建分镜节点
- `src/nodes/index.ts` — 注册 storyboard 类型
- `src/components/node-editor/NodePalette.tsx` — 面板新增分镜项
- `src/components/inspector/Inspector.tsx` — 新增 ScriptInspector / StoryboardInspector 组件

### 变更 2：时间线分镜点击行为——从直接生成改为分析+调出节点

**变更前**：点击时间线空片段 → 直接找画布第一个生成节点执行 → 生成结果填充到片段。

**变更后**：
```
点击空片段 → 分析 spec（角色ID/场景ID/物品/分镜）
            → 查找素材库中对应卡片
            → 自动在画布上创建角色卡/场景卡/物品卡/剧本/分镜节点
            → 通知面板列出调出的节点
            → 用户手动连线 → 手动运行生成节点
```

**新增能力**：
- 根据 `TrackClip.spec.characterIds` 自动创建角色卡节点
- 根据 `TrackClip.spec.sceneId` 自动创建场景卡节点和剧本节点
- 根据片段名称/对白匹配物品卡并创建节点
- 自动创建分镜节点（携带景别、描述、对白等）
- 已存在的节点不会重复创建

**函数变更**：`handleTriggerGenerate` → `handleAnalyzeClip`

**受影响文件**：
- `src/components/timeline/TimelineView.tsx` — ClipBlock props 改名、新增 handleAnalyzeClip

### 变更 3：提示词节点新增「刷新状态」按钮

**变更前**：提示词文本只能手动输入，无法一键收集上游信息。

**变更后**：
- 提示词节点检查器标题栏新增 **「刷新状态」** 按钮（RefreshCw 图标）
- 点击后自动遍历上游连线节点，收集信息并按格式写入：
  - 角色卡：`角色「名称」：外观描述`
  - 场景卡：`场景「名称」：场景描述`
  - 物品卡：`物品「名称」：描述`
  - 剧本节点：`[剧本·第N场 场次名]\n内容`
  - 分镜节点：`[分镜·镜N 景别]\n描述\n对白：「xx」`
- 用户可在自动填充的基础上二次编辑

**新增组件**：`PromptInspector`（从内联 JSX 提取为独立组件）

**受影响文件**：
- `src/components/inspector/Inspector.tsx` — 新增 PromptInspector、导入 RefreshCw/useCallback

## 五、测试建议

1. **剧本/分镜节点拆分**：
   - 从节点面板拖入剧本节点和分镜节点
   - 在灵感编辑器中先保存剧本场次和分镜
   - 在检查器下拉中选择，验证自动填充
2. **时间线分析**：
   - 导入分镜（含角色名/场景名）
   - 点击空片段，验证画布上自动出现对应节点
   - 手动连线后运行生成
3. **提示词刷新**：
   - 将角色卡/场景卡/剧本/分镜连线到提示词节点
   - 点击"刷新状态"，验证文本自动写入
   - 在自动文本基础上编辑
4. **配置导入导出**：
   - 导出包含密钥的配置 → 导入到另一环境 → 验证密钥已更新
   - 导出不含密钥的配置 → 导入 → 验证密钥保留原值
