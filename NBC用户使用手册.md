## 0. 先读这里（无门槛说明）

### 0.1 你需要先知道的 6 个词

- **项目**：一个创作“文件夹”。每个项目有自己的画布（节点/连线）、灵感（剧本/分镜/三卡）、时间线、素材关联。
- **节点**：一个功能模块（例如“角色卡”“提示词”“GPT 图像生成”“输出”）。节点之间可以连线。
- **连线**：数据流动的方向。上游节点把文字/图片/视频传给下游节点。
- **Provider**：一组“接口配置”。例如 GPT Image 2、Seedance、OSS、飞书云盘都属于 Provider，需要在“设置面板”里配置密钥和端点。
- **素材（Asset）**：图片/视频/文本卡片等资产，统一管理在“素材浏览器”。
- **时间线**：把多个分镜片段按时间顺序排成视频（还支持字幕、对话、BGM 轨道）。

### 0.2 NBC 能做什么（用一句话）

把“想法 → 剧本 → 分镜 → 角色/场景/物品卡 → 提示词 → 文生图/图生视频 → 素材管理 → 时间线 → 导出视频”串成一套可视化流程。

### 0.3 安全与数据位置

- NBC 是桌面应用（Electron）。绝大多数数据保存在本地（浏览器 localStorage + 本地素材目录）。
- API Key / Secret 会存本地。你可以在“设置 → 导出配置”时选择是否**包含密钥**。
- 生成结果可以同时保存到：本地 / OSS / 飞书（见第 19 章）。

## 1. 产品简介

NBC（Node-Based Creator）是一款**节点式素材创作器**。你通过拖拽节点、连线、点击运行，就能把上游信息自动组合成提示词并驱动下游模型生成。

### 核心能力

| 能力 | 说明 |
|------|------|
| 文生图 | GPT Image 2、Banana Image |
| 文生视频 / 图生视频 | Seedance 2.0（支持首帧/首尾帧/多模态/视频编辑/延长等模式） |
| 灵感生产 | 灵感编辑器：剧本/分镜/三卡 + AI 辅助 + 版本管理 |
| 素材管理 | 本地扫描 + OSS 拉取 + 飞书云盘同步 + 标签/项目归属 |
| 时间线 | 多轨道分镜编排 + 批量生成 + 导出脚本/导出视频 |
| 三路保存 | 输出节点一键同时保存到本地/OSS/飞书 |

## 2. 安装与启动

### 2.1 启动方式（开发/生产）

如果你是开发者：

```bash
npm run dev            # 纯 Web 模式，浏览器访问 http://localhost:5173/
npm run electron:dev   # Electron 桌面应用模式（推荐体验完整功能）
```

生产模式：

```bash
npm run electron:prod  # 构建后启动 Electron 应用
```

### 2.2 首次启动会发生什么

- 第一次启动会弹出“首次配置向导”（可跳过，后面随时能在设置面板里改）。
- 推荐先完成：GPT Image 2 / Seedance / LLM（提示词优化）/ OSS / 飞书云盘 的配置，否则相关功能会提示“未配置”。

## 5. 设置面板（含 OSS / 飞书云盘配置）

打开方式：顶部工具栏 → **API 设置**（齿轮）。

设置面板分为三块：

- **通用设置**：本地素材目录、操作人用户名、项目自动备份间隔
- **飞书多维表格同步**：内置飞书事件同步配置与执行（一键同步 + 自动定时）
- **API 配置**：所有 Provider（GPT/Seedance/LLM/OSS/飞书/自定义 Provider）

### 5.1 通用设置（每一项怎么用）

#### 5.1.1 默认本地素材目录 / 输出位置

- 你会看到一行路径（例如 `H:\素材库`），右侧有 **选择目录**。
- 点击 **选择目录**：弹出系统目录选择框，选定后：
  - “素材浏览器 → 扫描本地文件夹”的默认路径会更顺手
  - 输出节点保存到本地时也会以这里为主目录

#### 5.1.2 操作人用户名

- 用途：用于记录生成日志、飞书同步队列的“操作者”字段（便于团队区分是谁生成/同步的）。
- 建议填写：你的昵称或工号（例如 `NBC User`）。

#### 5.1.3 项目自动保存备份

- 用途：定时把项目导出为 `.nbc.json` 文件（类似“自动备份存档”）。
- 下拉选项：关闭 / 每 5 分钟 / 每 10 分钟 / 每 30 分钟 / 每 1 小时。

### 5.2 API 配置（Provider 面板通用操作）

你会看到一张张 Provider 卡片（可折叠），例如：

- GPT Image 2
- Banana Image
- Seedance 2.0
- LLM（提示词优化）
- 阿里云 OSS
- 飞书云盘

每个 Provider 卡片都支持以下按钮/控件：

| 控件/按钮 | 作用 |
|---|---|
| 展开/收起 | 点击卡片标题区域展开/折叠配置区 |
| 启用/禁用（Toggle） | 临时关闭某个 Provider（不删除配置） |
| 显示/隐藏密码（眼睛） | 密码字段可切换明文显示 |
| 测试连接 | 对默认端点发起 GET 请求，判断网络是否可达（部分服务返回 HTTP 错误也会视为“可达”） |
| 保存配置 | 把你输入的内容写入本地配置 |
| 移除 | 删除 Provider（一般不建议删除内置 Provider） |

### 5.3 OSS 配置（阿里云 OSS）

目标：让 NBC 能把生成结果上传到 OSS，并在素材浏览器里拉取 OSS 文件列表。

#### 5.3.1 你需要准备什么

- AccessKey ID
- AccessKey Secret
- Bucket Name（桶名）
- Region（例如 `oss-cn-shenzhen`）
- OSS Endpoint URL（通常类似 `https://oss-cn-shenzhen.aliyuncs.com`）

#### 5.3.2 在 NBC 里怎么填

1. 顶部工具栏 → **API 设置**
2. 找到 Provider：**阿里云 OSS**
3. 点击卡片展开
4. 依次填写：
   - AccessKey ID
   - AccessKey Secret
   - Bucket Name
   - Region
   - 端点 URL（如果你的 OSS Endpoint 不同）
5. 点击 **保存配置**
6. 点击 **测试连接**（可选）

#### 5.3.3 常见填法示例（仅示例）

- Region：`oss-cn-shenzhen`
- 端点 URL：`https://oss-cn-shenzhen.aliyuncs.com`

### 5.4 飞书云盘配置

目标：让 NBC 能从飞书云盘指定文件夹同步素材，并在“素材浏览器”里直接浏览/拖拽使用。

#### 5.4.1 你需要准备什么

- App ID（飞书应用 ID）
- App Secret（飞书应用密钥）
- 文件夹 Token（你要同步的飞书文件夹 token）

#### 5.4.2 在 NBC 里怎么填

1. 顶部工具栏 → **API 设置**
2. 找到 Provider：**飞书云盘**
3. 点击卡片展开
4. 依次填写：
   - App ID
   - App Secret
   - 文件夹 Token
   - 端点 URL（一般保持默认 `https://open.feishu.cn/open-apis`）
5. 点击 **保存配置**
6. 点击 **测试连接**（可选）

#### 5.4.3 配置后如何使用

见第 7 章「飞书云盘同步」：在素材浏览器里点击“飞书云盘”按钮即可拉取文件列表，并缓存到本地素材目录。

### 5.5 配置导入/导出（团队协作常用）

Provider 区域右上角有两个按钮：

- **导入**：从 JSON 文件导入 Provider 配置
- **导出**：导出 Provider 配置（会弹出"导出全部配置"窗口）

#### 5.5.1 导出配置

导出弹窗里有一个重要复选框：

- **包含 API Key 等密钥**
  - 勾选：导出文件包含所有密钥（API Key、AccessKey Secret、App Secret 等），适合**自己多台电脑同步配置**或**团队内完全信任的成员间迁移**
  - 不勾选：导出文件会把密钥字段置空，只保留端点 URL 等非敏感信息（适合给团队成员共享配置模板）

导出文件名为 `nbc_config_日期.json`，包含：
- 所有 Provider 配置（含端点、参数结构、启用状态）
- 通用设置（自动备份间隔等）
- 飞书同步配置（App ID、表格 Token、事件白名单、自动同步设置等）
- 如果勾选了密钥，则包含完整的 API Key / App Secret

#### 5.5.2 导入配置（会合并，不会覆盖未提供的密钥）

导入时，NBC 的合并策略是"**智能合并**"——**Provider 和飞书同步配置都遵循同一规则**：

1. **相同的 Provider ID 已存在**：
   - 如果导入的密钥字段为空，会**保留你本机已有的密钥**（不会用空值覆盖）
   - 如果导入的密钥字段有值，会用导入的密钥更新
   - 端点 URL 和参数结构会用导入的覆盖
2. **新的 Provider ID**（你本地没有的）：直接新增
3. **通用设置**（自动备份间隔等）：用导入的值覆盖

**举例说明**：
- 张三导出了一份"不包含密钥"的配置给李四
- 李四导入后，所有 Provider 结构都有了，但 API Key 保留李四自己的
- 如果张三导出时"包含了密钥"，李四导入后就会用张三的密钥（适合迁移到新电脑）

#### 5.5.3 导入操作步骤

1. 点击 **导入** 按钮
2. 在文件对话框中选中 `.json` 配置文件
3. 系统自动合并导入，并在通知面板显示"成功导入 X 个 Provider 配置、通用设置、飞书同步配置"

> **飞书同步配置导入规则**：如果导入的 `appSecret` 为空（导出时未勾选密钥），保留你本地的密钥；已填则有值覆盖。事件白名单、自动同步开关和间隔直接使用导入的值。

### 5.6 飞书多维表格同步（内置同步面板）

位于"通用设置"和"API 配置"之间，可以直接在 NBC 内完成飞书表格同步的全部操作。

#### 5.6.1 它能做什么

- 将 NBC 操作事件（项目创建、生成开始/完成/失败）自动写入飞书多维表格
- 支持**手动一键同步**和**自动定时同步**两种模式
- 配置保存在本地 localStorage，无需外部文件

#### 5.6.2 配置步骤

1. 在飞书开放平台创建应用，获取 App ID 和 App Secret
2. 在飞书中创建多维表格，记录 App Token 和 Table ID
3. 在表格中创建字段：`文本`（主键）、`事件类型`、`节点名称`、`状态`、`详细信息`、`操作人`
4. 在 NBC 设置面板的"飞书多维表格同步"中填入 4 个值
5. 选择要同步的事件类型（可多选）
6. （可选）勾选"启用自动同步"并设置间隔
7. 点击"立即同步"测试

#### 5.6.3 控件说明

| 控件 | 作用 |
|---|---|
| App ID / App Secret | 飞书应用凭据（密码框） |
| 多维表格 App Token | 目标 Base 的 Token |
| 表格 Table ID | 目标表的 ID |
| 同步事件类型按钮 | 切换哪些事件上报（项目创建/生成开始/完成/失败） |
| 启用自动同步 | 勾选后 NBC 按间隔自动同步 |
| 间隔下拉 | 1/5/10/30/60 分钟 |
| 立即同步 | 手动执行一次同步，结果直接显示在下方 |
| 结果条 | 绿色=成功（含同步条数），红色=失败 |

#### 5.6.4 工作原理

```
用户在 NBC 中操作 → emitNBCEvent() 写入 nbc_events.jsonl
                                       ↓
                     点击"立即同步" 或 自动定时器触发
                                       ↓
                     调用 Electron 主进程 IPC feishu:sync:run
                                       ↓
                     读取增量事件 → 飞书 API 批量写入 → 更新 cursor
```

> **注意**：此功能需在 Electron 桌面应用中使用。Web 开发模式会提示"无法运行同步"。

| 悬停 🗑️ | 删除 |

## 7. 素材浏览器（素材库）

素材浏览器用于统一管理：本地素材、OSS 素材、飞书云盘素材、以及各种文本卡片资产。

### 7.1 顶部按钮与开关

| 按钮/控件 | 用途 |
|---|---|
| 批量选择 | 进入多选模式（素材卡片左上角出现勾选框） |
| 网格/列表视图 | 两种展示方式切换 |
| 扫描本地文件夹 | 导入本地文件夹中的图片/视频 |
| OSS | 从 OSS 拉取远程素材列表 |
| 飞书云盘 | 从飞书云盘指定文件夹同步文件 |
| 全选当前 / 取消全选 | 多选模式下快速选择 |
| 删除 / 取消 | 多选模式下批量删除或退出 |

### 7.2 筛选与搜索

- **按分类**：按标签过滤（角色卡/场景卡/GPT Image/Seedance/输出等）。
- **按项目**：按项目归属过滤（全部/未分配/某个项目）。
- **搜索框**：按素材名称模糊搜索。

### 7.3 素材卡片（单个素材的操作）

| 操作 | 方式 |
|---|---|
| 选中 | 单击卡片 |
| 全屏查看 | 双击缩略图 |
| 拖到画布 | 拖拽到画布会自动创建对应节点（图片/视频 → 素材输入节点；文本卡 → 对应卡片节点） |
| 编辑标签 | 悬停点击 ✏️ |
| 删除素材 | 悬停点击 🗑️（OSS 素材会尝试删除远端） |

### 7.4 本地文件夹扫描

1. 素材浏览器工具栏点击 **扫描本地文件夹**
2. 选择一个目录
3. NBC 会扫描其中的图片/视频并加入素材库

## 8. 灵感编辑器（剧本/分镜/三卡）

灵感编辑器是“从文字到结构化素材”的核心面板。它有 3 步：

1. 第 1 步：剧本
2. 第 2 步：分镜
3. 第 3 步：三卡（角色卡/场景卡/物品卡）

并且支持：

- 自由文本编辑（默认）
- 结构化编辑（把剧本解析为场次卡、分镜解析为镜头卡、三卡解析为表单）
- AI 对话辅助（右侧聊天）
- 版本历史（记录/恢复/删除）
- 一键提取三卡、一键生成角色形象

### 8.1 顶部“步骤”与“子标签”

| 你会看到 | 作用 |
|---|---|
| 第1步：剧本 / 第2步：分镜 / 第3步：三卡 | 切换当前编辑的内容类别 |
| 角色卡 / 场景卡 / 物品卡 | 仅在“三卡”步骤下出现，用于切换具体卡片类型 |

### 8.2 编辑模式：自由文本 vs 结构化

灵感编辑器顶部会有“模式切换”按钮：

- **自由文本**：直接写一段文字（更像写文档）
- **结构化**：把内容拆成可编辑的结构（更像填表/卡片）

#### 8.2.1 剧本/分镜的结构化按钮

| 按钮 | 用途 |
|---|---|
| 解析为卡片（剧本/分镜） | 把当前文本解析成“场次/镜头”卡片结构 |
| 写回文本（剧本/分镜） | 把结构化内容再拼回一段完整文本写入“当前版” |

#### 8.2.2 三卡的结构化按钮

| 按钮 | 用途 |
|---|---|
| 解析为卡片（三卡） | 把当前文本解析成角色/场景/物品的结构化表单 |
| 写回文本（三卡） | 把结构化表单再导出为文本写入“当前版” |

### 8.3 三卡增强（常用按钮）

| 按钮 | 用途 |
|---|---|
| 一键提取三卡 | 从当前剧本 + 分镜中自动提取角色/场景/物品信息，并入库到素材库 |
| 一键生成角色形象 | 对素材库内的角色卡批量生成角色参考表（PNG），并展示进度 |
| 从素材库选择 | 从素材库选择一条资产加载到当前分类 |
| 保存当前 | 把当前类别内容保存/更新到素材库（通常生成文本卡） |
| 全部保存 | 一次性保存“剧本 + 分镜 + 三卡”到素材库 |

### 8.4 版本与历史（防止写崩）

| 控件 | 用途 |
|---|---|
| 自动记录（下拉） | 设置自动保存版本的间隔（关闭/1/5/10/30 分钟） |
| 手动记录 | 立刻保存一个版本快照 |
| 版本历史 (N) | 打开侧边栏，查看版本列表 |
| 恢复此版本 | 把某个历史版本恢复为当前内容 |
| 删除记录 | 删除某条版本记录 |

### 8.5 角色卡里的“角色形象”区域（生成参考表）

当你在“三卡 → 角色卡”时，会看到和图片相关的按钮：

| 按钮 | 用途 |
|---|---|
| 上传参考图 | 选择本地图片作为参考 |
| 从素材库绑定 | 从素材库选一张图绑定为参考图 |
| 垃圾桶（清空） | 清空参考图 |
| 生成角色参考表 | 调用 GPT Image 2 生成“角色参考表”图 |
| 保存图片到素材库 | 把生成结果保存为素材库图片资产，并绑定到卡片封面 |
| 更换绑定 | 给已生成图像重新绑定素材库图片 |

### 8.6 右侧聊天面板（AI 辅助）

| 按钮/动作 | 用途 |
|---|---|
| 历史 (N) | 打开会话列表 |

| 删除会话 | 删除某个会话 |

## 9. 节点编辑器（画布/节点/检查器）

节点编辑器由 3 个部分组成：

- **画布（Flow）**：拖拽节点、连线、运行
- **节点面板（Node Palette）**：可用节点列表
- **属性检查器（Inspector）**：编辑选中节点的参数

### 9.1 画布工具栏（FlowEditor）按钮说明

| 按钮 | 用途 | 典型场景 |
|---|---|---|
| 保存项目 | 保存当前画布到当前项目 | 画布改动后手动保存 |
| 导出文件 | 导出 `.nbc.json` | 分享给别人/备份 |
| 导入文件 | 导入 `.nbc.json`（覆盖当前画布） | 拿到别人的工作流直接用 |
| 全部运行 | 运行画布内所有生成节点（会弹确认） | 一键跑完整条流水线 |
| 删除 | 删除选中的节点/连线 | 清理画布 |

左侧状态区会显示：

- 当前项目名
- 保存状态点：🟠（未保存）/ 🟢（已保存）
- 最近保存时间

### 9.2 画布基础操作

| 操作 | 方式 |
|---|---|
| 平移画布 | 鼠标中键拖拽 / 按住空白区域拖拽 |
| 缩放画布 | 鼠标滚轮 / 右下角 Controls 的 +/- |
| 自适应（fit view） | Controls 的自适应按钮 |
| 框选多个节点 | Shift + 拖拽空白区域 |
| 删除选中 | Backspace / Delete |
| 小地图 MiniMap | 右下角显示全局位置，支持快速定位 |

### 9.3 节点面板（Node Palette）

节点面板按类别折叠：

- 卡片：角色卡、场景卡、物品卡
- 文字：剧本、分镜、提示词
- 生成：GPT 图像生成、Banana 图像生成、Seedance 视频生成
- 输出：输出

添加节点的方式：

- **点击每个节点右侧的 “+”**：立即在画布生成一个节点
- **拖拽节点条目到画布**：在鼠标落点处生成节点

### 9.4 属性检查器（Inspector）

选中一个节点后，右侧检查器会显示该节点的所有可编辑字段。

| 按钮/动作 | 用途 |
|---|---|
| 删除节点 | 在检查器标题栏里删除当前选中节点 |
| 编辑字段 | 实时写入节点参数（会触发自动保存） |

### 9.5 节点类型逐个详解（每个端口/按钮怎么用）

下面按“你在画布上能看到的节点”逐个说明。

#### 9.5.1 素材输入节点（Asset Input）

用途：把“素材库里的图片/视频”作为下游节点的参考图/参考视频输入。

- **如何创建**：从素材浏览器拖拽一张图/一个视频到画布，会自动生成素材输入节点。
- **输出端口**：右侧一个输出点（供下游连接）。
- **检查器字段**：素材 ID（只读）。

常见连法：

- 素材输入 → GPT 图像生成节点的 `image`
- 素材输入 → Seedance 节点的 `image`（首帧/首尾帧）或 `video`（视频编辑/多模态）

#### 9.5.2 角色卡节点（Character Card）

用途：描述“角色长什么样”，并可携带一致性参数供下游引用。

关键字段（检查器）：

- 名称、外观描述、参考图片路径
- 多角色（逗号分隔 ID）
- 一致性：面部一致性提示词、体型/着装一致性、负面提示词、推荐种子、参考图 URLs
- TTS：音色 ID、语速、音调（用于对白配音扩展）

输出端口：右侧输出点（提供角色文本给下游提示词节点）。

#### 9.5.3 场景卡节点（Scene Card）

用途：描述“在哪儿发生”，例如环境、光照、氛围。

关键字段：

- 名称、场景描述、参考图片路径
- 灵感库：快速选择预设场景

输出端口：右侧输出点。

#### 9.5.4 物品卡节点（Item Card）

用途：描述道具/物品外观（例如“银色冲锋枪”“医疗箱”）。

关键字段：

- 名称、物品描述、参考图片路径
- 灵感库：快速选择预设物品

输出端口：右侧输出点。

#### 9.5.5 剧本节点（Script）

用途：代表剧本中的一场戏（场次），从灵感编辑器的结构化剧本数据中选择或自由填写文本。

关键字段：

- **灵感编辑器场次**：下拉选择框，自动从灵感编辑器「第1步：剧本」读取已保存的结构化场次列表。选择后自动填充该场次的内容
- 剧本内容：可手动编辑，也可从灵感编辑器场次自动填充

输出端口：右侧输出点（为下游提示词节点提供剧本上下文）。

#### 9.5.6 分镜节点（Storyboard）

用途：代表一个分镜镜头，从灵感编辑器的结构化分镜数据中选择或自由创建。

关键字段：

- **灵感编辑器分镜**：下拉选择框，自动从灵感编辑器「第2步：分镜」读取已保存的分镜镜头列表。选择后自动填充镜头号、景别、描述、对白、关联角色/场景/物品等信息
- 选好分镜后，下方信息栏显示：镜头号、景别、对白预览、描述

特殊功能：
- 分镜节点携带**角色 ID 列表**（`storyboardCharacterIds`）、**场景 ID**（`storyboardSceneId`）、**物品 ID 列表**（`storyboardItemIds`），时间线分析时会用这些信息自动调出对应的卡片节点

输出端口：右侧输出点。

#### 9.5.7 提示词节点（Prompt）★ 新增「刷新状态」按钮

用途：把上游角色/场景/物品/剧本/分镜信息组合成最终提示词，支持一键刷新。

端口：

- 左侧：输入端口（接上游卡片/剧本/分镜）
- 右侧：输出端口（接生成节点的 `prompt`）

关键字段：

- 提示词文本：支持手动编辑，也支持**一键刷新**
- **「刷新状态」按钮**：点击后自动遍历上游连线节点，收集角色卡、场景卡、物品卡、剧本、分镜的信息，自动写入提示词文本框（格式为 `角色「名称」：描述` / `场景「名称」：描述` / `[剧本·第N场 场次名]` / `[分镜·镜N 景别]` 等）。你可以在此基础上二次编辑
- AI 提示词优化器：见第 10 章

#### 9.5.8 GPT 图像生成节点（GPT Image 2）

用途：把提示词变成图片。

端口：

- `prompt`（左侧 25%）：接提示词节点
- `image`（左侧 55%）：接素材输入（参考图）
- `output`（右侧）：接输出节点

按钮：

- 右上角 **生成按钮**：图标随状态变化
  - ▶️ 空闲：点击开始生成
  - 🔄 运行中：禁用
  - ✅ 完成：生成成功
  - ❌ 失败：失败（可再次点击重试）

常用字段：

- 模型：`gpt-image-2-vip` / `gpt-image-2`
- 画面比例/分辨率、图片质量
- 参考图 URL（也可以通过连线传入）
- 并发数量（一次生成多张）
- 风格选择（从素材库加载）

#### 9.5.9 Banana 图像生成节点（Banana Image）

用途：另一条文生图通道（同样支持参考图）。

端口：

- `prompt`、`image`、`output`（与 GPT Image 类似）

按钮：

- 右上角“生成”按钮（同样是 ▶️/🔄/✅/❌ 状态）

常用字段：

- bananaAspectRatio（例如 `1024x1024`）
- bananaModel（默认 `gpt-image-2`）

#### 9.5.10 Seedance 视频生成节点（Seedance 2.0）

用途：文本/图片/视频驱动生成视频。

端口：

- `prompt`（左侧 20%）：接提示词
- `image`（左侧 45%）：接素材输入（首帧/参考图）
- `video`（左侧 70%）：接素材输入（参考视频）
- `output`（右侧）：接输出节点

按钮：

- 右上角“运行”按钮：▶️/🔄/✅/❌

关键字段（检查器）：

- 模型：Seedance 2.0 / Seedance 2.0 Fast
- 生成模式：文生视频 / 图生视频·首帧 / 图生视频·首尾帧 / 多模态参考 / 视频编辑 / 视频延长
- 分辨率、宽高比、时长
- 推理模式：在线 / 离线（flex）
- 生成有声视频、返回尾帧图、启用联网搜索
- 并发数量、参考素材数量规划
- 风格选择

#### 9.5.11 输出节点（Output）

用途：定义生成结果怎么保存（本地/OSS/飞书），并提供“打开”入口。

端口：

- 左侧 `input`：接任意生成节点（GPT Image / Banana / Seedance）

关键字段（检查器）：

- 保存到本地（复选框）
- 上传到 OSS（复选框）
- 同步到飞书（复选框）

画布上的按钮：

- **打开**：当节点已有本地路径或 OSS URL 或结果 URL 时可点，打开文件/链接。

## 10. AI 提示词优化

AI 提示词优化器位于：**提示词节点（Prompt）→ 右侧检查器**。

### 10.1 你会看到的按钮

| 按钮 | 用途 |
|---|---|
| AI 优化提示词 | 读取上游上下文（角色/场景/物品/剧本），调用 LLM 生成更好的提示词 |
| 采用 | 把优化后的提示词写回节点 |
| 再试 | 重新优化一次 |
| 放弃 | 关闭优化结果，不修改原文本 |

### 10.2 常见用法

1. 先把角色卡/场景卡/剧本节点连到提示词节点
2. 在提示词节点写一个“粗提示词”（哪怕很短也行）
3. 点击 **AI 优化提示词**
4. 对比结果后点击 **采用**

- 会弹出确认框
- 按拓扑顺序依次执行所有生成节点

### 13.3 片段操作（Clip）★ 新行为：点击空片段不再直接生成

| 裁剪起止 | 拖拽片段左右边缘 |

片段状态颜色：

- empty：待生成
- generating：生成中
- done：已完成
- failed：失败

#### 13.3.1 点击空片段发生了什么（片段分析功能）

点击时间线上的空片段时，NBC 不会直接生成，而是会**分析该片段在项目内关联了哪些卡片和节点**：

1. 读取片段的 `spec`（角色 ID 列表、场景 ID、分镜信息等）
2. 在素材库中查找对应的角色卡、场景卡、物品卡
3. 检查画布上是否已有这些卡片对应的节点
4. **自动在画布上创建缺失的节点**（角色卡节点、场景卡节点、物品卡节点、剧本节点、分镜节点）
5. 通知面板弹出分析结果，列出已调出的节点清单
6. 你在画布上将这些节点**连线**（角色卡→场景卡→物品卡→剧本→分镜→提示词→生成节点→输出节点）
7. 连线完成后，点击生成节点的 **▶️ 运行** 生成素材

> **提示**：只有已存在于时间线片段 `spec` 中的角色/场景/物品卡片才会被调出。如果片段没有关联任何卡片，建议先在片段详情中编辑（悬停展开后设置景别、角色等），或手动从素材库拖入卡片。

## 19. 三路保存机制（本地/OSS/飞书）

三路保存的入口在：**输出节点（Output）**。

### 19.1 输出节点的 3 个复选框

| 选项 | 结果 |
|---|---|
| 保存到本地 | 生成结果落到本地素材目录，并写入输出节点显示路径 |

### 19.2 常见问题提示

### 19.3 NBC 工作状态上传飞书表格（Agent 定时同步脚本）

除了输出节点直接上传文件到飞书云盘，NBC 还提供一套**将工作状态自动上报到飞书多维表格**的机制，适合团队场景：成员在 NBC 里的每次生成、项目创建、工作流导出等操作，都会被记录到一个飞书表格里，方便团队追踪进度。

**两种使用方式**：

| 方式 | 适合场景 | 说明 |
|---|---|---|
| **NBC 内置面板**（推荐） | 个人使用、无需外部依赖 | 在设置面板直接配置 + 一键同步 + 自动定时同步 |
| **独立脚本** | CI/服务器、高级用户 | `node scripts/feishu-sync.js` + 配置文件/环境变量

#### 19.3.1 工作原理

```
用户在 NBC 中操作（生成/创建项目/导出等）
    ↓
emitNBCEvent() 写入 nbc_events.jsonl（事件日志文件）
    ↓
feishu-sync.js（Agent 脚本，建议每 5 分钟定时执行）
    ↓
读取增量事件 → 调用飞书多维表格 API → 批量插入记录
```

#### 19.3.2 相关文件说明

| 文件/脚本 | 位置 | 用途 |
|---|---|---|
| `nbc_events.jsonl` | `H:\素材库\数据日志\` | NBC 运行时自动生成的事件日志（JSONL 格式，每行一条） |
| `f eishu_queue.jsonl` | `H:\素材库\数据日志\` | 飞书同步队列文件（输出节点勾选"同步到飞书"时写入） |
| `feishu-sync.js` | `scripts/feishu-sync.js` | Agent 定时脚本：读取 `nbc_events.jsonl`，增量同步到飞书多维表格 |
| `nbc_feishu_sync_config.template.json` | `scripts/` | 配置文件模板：复制到 `文档\NBC素材\` 后填自己的飞书信息 |
| `test-feishu-bitable.js` | `scripts/test-feishu-bitable.js` | 测试脚本：插入一条测试记录到飞书多维表格，验证配置和权限 |
| `test-feishu.js` | `scripts/test-feishu.js` | 测试脚本：验证飞书 Open API 连接和文件夹读取 |

#### 19.3.3 方式一：NBC 内置面板（推荐）

从设置面板的"飞书多维表格同步"区块直接配置和同步，无需外部脚本或定时任务。

**面板位置**：顶部工具栏 → API 设置 → 滚动到"飞书多维表格同步"

**面板包含的控件**：

| 控件 | 说明 |
|---|---|
| App ID | 飞书应用的 App ID |
| App Secret | 飞书应用的 App Secret（密码框） |
| 多维表格 App Token | 目标 Base 的 App Token |
| 表格 Table ID | 目标表的 Table ID |
| 同步事件类型 | 4 个可切换按钮：项目创建、生成开始、生成完成、生成失败 |
| 启用自动同步 | 勾选后按间隔自动同步 |
| 自动同步间隔 | 1/5/10/30/60 分钟 |
| 立即同步 | 手动触发一次同步 |

**操作步骤**：

1. 在飞书开放平台创建应用，获取 App ID / App Secret
2. 在飞书中创建多维表格，记录 App Token / Table ID
3. 在表格中创建字段：`文本`、`事件类型`、`节点名称`、`状态`、`详细信息`、`操作人`
4. 打开 NBC 设置面板，找到"飞书多维表格同步"
5. 填入 4 个飞书值，点击要同步的事件类型
6. （可选）勾选"启用自动同步"并设置间隔
7. 点击"立即同步"测试

**自动同步说明**：

- 勾选"启用自动同步"后，NBC 运行期间会按设定间隔自动执行同步
- 关闭 NBC 窗口后自动同步会暂停（下次启动需重新开启面板勾选）
- 结果会通过通知面板静默提示（只在新事件数 > 0 时弹出）

#### 19.3.4 方式二：独立脚本 feishu-sync.js（高级用户/CI 环境）

独立脚本支持 **3 层配置**，从高到低的优先级为：

1. 命令行参数 → `node scripts/feishu-sync.js D:\my_config.json`
2. 环境变量 → `NBC_FEISHU_APP_ID` 等
3. 配置文件 → `文档\NBC素材\nbc_feishu_sync_config.json`

**推荐方式：创建个人配置文件（无需改动脚本）**

1. 复制配置模板到你的数据目录：

   ```
   copy scripts\nbc_feishu_sync_config.template.json "%USERPROFILE%\Documents\NBC素材\nbc_feishu_sync_config.json"
   ```

2. 用记事本打开 `nbc_feishu_sync_config.json`，填入你的飞书信息：

   ```json
   {
     "feishuAppId": "cli_你的AppId",
     "feishuAppSecret": "你的AppSecret",
     "bitableAppToken": "你的多维表格Token",
     "bitableTableId": "你的表格TableId"
   }
   ```

3. 保存后运行 `node scripts/feishu-sync.js`，脚本会自动读取配置文件

配置文件**可自定义的全部项**：

| 配置项 | 必填 | 说明 | 示例 |
|---|---|---|---|
| `feishuAppId` | ✅ | 飞书应用的 App ID | `cli_a97f...` |
| `feishuAppSecret` | ✅ | 飞书应用的 App Secret | `idXvoi...` |
| `bitableAppToken` | ✅ | 飞书多维表格 App Token | `Wg8Jb...` |
| `bitableTableId` | ✅ | 表格的 Table ID | `tblHwf...` |
| `nbcDir` | | NBC 数据目录（默认：`文档/NBC素材`） | 可改为素材库路径 |
| `batchSize` | | 单次批量插入最大条数 | `100`（默认） |
| `allowedActions` | | 要同步的事件白名单 | 见下方数组格式 |
| `fieldNames` | | 飞书表格列名映射 | 如果你的列名不同，在这里改 |

**自定义白名单**（在配置文件的 `allowedActions` 中增减）：

```json
"allowedActions": [
  "project:create",
  "generation:start",
  "generation:complete",
  "generation:fail"
]
```

**自定义表格列名映射**（如果你的飞书表格列名和默认不同）：

```json
"fieldNames": {
  "primary": "文本",
  "eventType": "事件类型",
  "nodeName": "节点名称",
  "status": "状态",
  "details": "详细信息",
  "operator": "操作人"
}
```

**使用环境变量（适合 CI/生产环境，更安全）**：

```bash
set NBC_FEISHU_APP_ID=cli_你的AppId
set NBC_FEISHU_APP_SECRET=你的AppSecret
set NBC_FEISHU_BITABLE_APP_TOKEN=你的多维表格Token
set NBC_FEISHU_BITABLE_TABLE_ID=tblH...
node scripts/feishu-sync.js
```

> **提示**：如果配置缺失，脚本运行时会打印清晰的指引信息，并告知如何修复。

脚本运行流程：
1. 读取 cursor 文件，获取上次同步到的行号
2. 从 `nbc_events.jsonl` 新增的行中读取事件
3. 白名单过滤（只同步 `allowedActions` 中的事件，默认为 `project:create`、`generation:start/complete/fail`）
4. 映射为飞书表格字段（列名按 `fieldNames` 映射）
5. 批量插入表格
6. 成功则更新 cursor，失败则不更新（下次重试）

#### 19.3.5 团队成员各自配置飞书表格同步的步骤

   - 打开“项目自动保存备份”（例如每 10 分钟）
3. 在 **API 配置**：
   - GPT Image 2：填 API Key，保存，测试连接（可选）
   - Seedance 2.0：填火山方舟 API Key，保存
   - LLM：填端点+API Key+model（用于提示词优化），保存
   - 阿里云 OSS：按第 5.3 填好并保存
   - 飞书云盘：按第 5.4 填好并保存
4. （可选）点击 **导出**，导出一份“不含密钥”的配置给团队成员使用

### 20.2 创建项目并套用模板

1. 打开 **项目管理** 面板，点击 **+ 新建项目**，命名：`巴别塔短片`
2. 顶部工具栏打开 **模板市场**
3. 选择 **完整创作工作流**，点击 **使用**
4. 返回画布，你会看到节点已经摆好并连线

提示：如果你不小心把面板拖乱了，点顶部的 **恢复默认布局**。

### 20.3 用灵感编辑器产出剧本/分镜/三卡，并入库素材

1. 打开 **灵感编辑器**
2. 在 **第1步：剧本** 写入（示例，随便写也行）：

```
蜂医在巴别塔大厅发现自毁装置即将启动，疾风冲入制止。二人争执后决定联手解除。
```

3. 在 **第2步：分镜** 写入（每行一镜头）：

```
全景·巴别塔大厅警报闪烁，蜂医抬头看向倒计时 6秒 淡入
中景·疾风冲入画面「停下！你会毁掉所有人」 5秒 硬切
特写·蜂医手指停在按钮上，冷静转身「那就一起把它关掉」 4秒 叠化
```

4. 在 **第3步：三卡**：
   - 点 **一键提取三卡**（从剧本/分镜自动提取角色/场景/物品）
   - 进入 **角色卡** 子标签，选中蜂医/疾风，补充外观描述（可选）
   - （可选）点击 **生成角色参考表** 为角色生成统一参考图
5. 点击 **全部保存**：把剧本/分镜/三卡写入素材库（后面可以拖到画布）

### 20.4 用素材浏览器整理素材（本地/OSS/飞书）

1. 打开 **素材浏览器**
2. 点击 **扫描本地文件夹**，把你已有的参考图/参考视频导入（可选）
3. 点击 **OSS**：拉取 OSS 文件列表（验证配置是否正确）
4. 点击 **飞书云盘**：同步飞书文件夹（验证配置是否正确）
5. 找到一张参考图（可选），拖到画布生成一个 **素材输入节点**

### 20.5 组装提示词：提示词库 + AI 优化

1. 顶部打开 **提示词库**
2. 选择一个“镜头/光影”模板，点击 **插入**（已复制到剪贴板）
3. 回到画布，选中 **提示词节点**，把剪贴板内容粘进去，并加一句你的需求，例如：

```
在 {{scene}} 中，{{character}} 正在阻止自毁装置启动。电影感，紧张氛围。
（此处粘贴提示词库里的镜头/光影模板）
```

4. 在右侧检查器点击 **AI 优化提示词**
5. 觉得满意就点 **采用**

### 20.6 生成图像与视频，并开启三路保存

1. 选中 **输出节点**，在检查器勾选：
   - 保存到本地 ✅
   - 上传到 OSS ✅
   - 同步到飞书 ✅
2. 在画布上先点击 **GPT 图像生成节点** 的 ▶️，生成关键帧图
3. 生成成功后，再点击 **Seedance 视频生成节点** 的 ▶️，基于提示词+参考图生成视频
4. 输出节点会显示：
   - 本地路径（💾）
   - OSS URL（☁️）
   - 并且有 **打开** 按钮可直接打开文件/链接

### 20.7 生成队列与失败日志排查（一定要会）

1. 打开 **生成队列**
   - 查看任务是否 running/completed/failed
   - 运行中的任务可点“取消”
2. 如果失败：
   - 打开 **失败日志**
   - 复制单条报告或导出全部日志发给开发排查
3. 修复配置后，回到节点点击 ▶️ 重试

### 20.8 把分镜导入时间线并批量生成

1. 打开 **时间线**
2. 点击 **导入分镜**
3. 点击 **从灵感编辑器导入**（直接导入第 2 步分镜）
4. 此时视频轨会出现 3 个片段（空坑位）
5. 点击 **批量生成**：系统会依次触发画布里的生成节点填充空片段
6. 生成完成后，片段状态会变为 done

### 20.9 导出视频（两种方式都走一遍）

方式 A（时间线内置脚本导出）：

1. 时间线点击 **导出视频**
2. 点击 **导出脚本**
3. 下载 `nbc_export_script.txt`，按脚本提示运行 ffmpeg 合并

方式 B（视频导出面板）：

1. 顶部工具栏点击 **视频导出**
2. 选择分辨率、fps、输出路径
3. 点击 **导出**
4. 完成后点 **打开文件夹**

### 20.10 备份与分享（项目/配置）

- 在画布工具栏点击 **导出文件**：得到 `.nbc.json`（项目工作流分享）
- 在设置面板点击 **导出**：得到 `nbc_config_YYYY-MM-DD.json`（Provider 配置分享）

## 22. 常见问题

### 22.1 点击生成提示“未配置 API Key”

- 到：顶部工具栏 → API 设置
- 找到对应 Provider（GPT Image 2 / Seedance / LLM），展开填好 API Key，点保存

### 22.2 OSS/飞书同步按钮点了没反应

- 先确认 Provider 已启用（Toggle 是打开状态）
- 点“测试连接”确认网络可达
- 检查 AccessKey/Secret、AppId/AppSecret 是否正确

### 22.3 导入模板/工作流后画布被覆盖了怎么办

- 模板“使用”会覆盖当前画布，这是设计如此
- 如果你之前导出过 `.nbc.json`，可以再导入回来
- 建议养成习惯：重要改动前先“导出文件”备份

### 22.4 时间线导出没有直接生成 MP4

- 时间线的“导出视频”主要生成 ffmpeg 拼接脚本
- 想直接导出 MP4：用顶部工具栏的“视频导出”面板（Electron 环境下支持）
