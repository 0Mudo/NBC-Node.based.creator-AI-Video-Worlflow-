# NBC 节点式 AI 多模态内容创作系统 · 用户手册

> 版本：V1.0.5 | 适用平台：Windows 10/11, macOS 12+

---

## 第一章 软件概述

### 1.1 软件简介

NBC（Node-Based Creator）是一款基于可视化节点流程图驱动多模态 AI 大模型协同生成图像和视频内容的桌面应用。用户无需编写代码，通过在画布上拖拽和连接 11 种功能节点，即可构建完整的多步骤 AI 创作工作流——从角色设定、场景定义，到提示词编排、AI 生成，再到结果保存，全部以可视化方式完成。

与传统的 AI 图像生成工具（如 Stable Diffusion WebUI、Midjourney）不同，NBC 的核心差异在于：(1) 采用有向无环图（DAG）描述创作流程，支持多步生成串联；(2) 自动收集上游节点信息填充提示词模板变量；(3) 内置时间线分镜系统，将生成素材组织和导出为视频分镜表；(4) 支持本地素材库管理，可拖拽素材直接接入工作流。

### 1.2 技术架构

NBC 采用三层架构设计：

- **渲染进程**（React 18 + TypeScript 5）：负责用户界面渲染、节点画布交互（ReactFlow 11）、状态管理（Zustand 4）、TailwindCSS 3 样式系统
- **预加载桥**（Electron contextBridge）：安全暴露文件操作 API，隔离渲染进程与 Node.js 能力
- **主进程**（Electron 29）：处理文件系统扫描（fs 模块）、API 代理请求（net.request）、`.nbc` 自定义协议（protocol.handle）、窗口管理

应用无后端服务器依赖，所有数据持久化通过 localStorage（项目数据 + 素材库索引）+ 本地文件系统（导出/导入 `.nbc.json` + 生成结果保存）完成。AI 模型调用通过 Vite 开发代理（Web 模式）或 Electron IPC 代理（桌面模式）转发至外部 API（GrsAI / 火山方舟 Ark / ComfyUI）。

### 1.3 运行环境要求

| 项目 | 最低要求 | 推荐配置 |
|------|----------|----------|

| CPU | 双核 2.0GHz | 四核 3.0GHz+ |
| 内存 | 4GB RAM | 8GB RAM+ |

### 1.4 安装与启动

**Windows**：
1. 下载 `NBC-Setup-1.0.5.exe` 安装包
2. 双击运行，按安装向导完成安装
3. 桌面快捷方式启动，或从开始菜单启动

**macOS**：
1. 下载 `NBC-1.0.5-arm64.dmg`（Apple Silicon）或 `NBC-1.0.5-x64.dmg`（Intel）
2. 双击挂载，将 NBC 拖入 Applications 文件夹
3. 首次打开时，如系统提示"来自未认证的开发者"，进入「系统偏好设置 → 安全性与隐私」点击"仍要打开"

**首次启动配置向导**：
首次运行 NBC 会弹出配置向导，要求输入 AI 服务的 API Key（GPT Image 2 的 GrsAI Key、Seedance 的火山方舟 Ark Key、ComfyUI 的本地地址）。可在后续通过「设置面板」修改。

**开发模式**（源码运行）：
```bash
npm install          # 安装依赖
npm run dev          # Web 模式启动（http://localhost:5173）
npm run electron:dev # Electron 桌面模式启动
```

## 第三章 节点系统

NBC 提供 11 种节点类型，分为四大类别：

### 3.1 节点类型总览

| 节点类型 | 类别 | 入口颜色 | 出点数 | 入点数 | 功能 |
|----------|------|----------|--------|--------|------|
| 素材输入 | 输入 | 青色 | 1 | 0 | 引用本地图片/视频素材 |
| 角色卡 | 卡片 | 红色 | 1 | 0 | 定义角色外貌与一致性约束 |
| 场景卡 | 卡片 | 金色 | 1 | 0 | 定义场景视觉环境 |
| 物品卡 | 卡片 | 橙色 | 1 | 0 | 定义关键道具设计 |
| 剧本 | 文字 | 青色 | 1 | 0 | 输入剧本/场次文本 |
| 分镜 | 文字 | 珊瑚色 | 1 | 0 | 定义镜头类型/对白/转场 |
| 提示词 | 文字 | 紫色 | 1 | 1 | 输入提示词、收集上游信息 |
| GPT图像生成 | 生成 | 粉色 | 1 | 1 | 调用 GPT Image 2 生成图像 |
| Banana图像生成 | 生成 | 橙色 | 1 | 1 | 调用 Banana 模型生成图像 |
| Seedance视频生成 | 生成 | 紫色 | 1 | 1 | 调用 Seedance 2.0 生成视频 |
| 输出 | 输出 | 珊瑚色 | 0 | 1 | 配置结果保存方式 |

### 3.2 素材输入节点

**用途**：引用本地文件夹中的图片或视频文件，作为 AI 生成的视觉参考。

**操作**：
1. 从素材库拖拽素材文件到画布空白区，自动创建素材输入节点（也可从节点面板点击添加后手动设置）
2. 节点标题栏右侧显示类型标签：🟢「图片」或 🟣「视频」
3. 当素材节点连线到提示词节点时，标题栏右侧自动显示变量标签（如「图片参考1」）——这是该素材在提示词中被引用的变量名

**属性**（检查器面板）：
- 标签：节点标题，默认显示素材文件名
- 素材 ID：关联的素材唯一标识

### 3.3 角色卡节点

**用途**：定义角色外貌特征、一致性约束，支持单角色或多角色配置。

**属性**：
- 角色名：单角色名字
- 角色外观描述：文字描述角色的外貌特征
- 角色多维数组：支持逗号分隔的多角色名，对应多条外观描述
- 角色参考图：角色的视觉参考图片路径
- 面部提示词：在生成时强制追加的面部一致性约束
- 体型/着装提示词：体型和服装的一致性约束
- 负面提示词：不希望出现在角色身上的特征
- 一致性种子：推荐的随机种子值，用于多张图片间保持角色一致
- TTS 语音 ID / 语速 / 音调：预留音频相关参数

**使用方式**：
- 角色卡 → 连线 → 提示词节点，提示词中可使用 `{{character}}` 变量自动替换为角色名

### 3.4 场景卡节点

**用途**：定义场景视觉环境，支持单场景或多场景配置。

**属性**：
- 场景名：单场景名字
- 场景描述：文字描述场景的环境特征（光照、色调、空间结构等）
- 场景多维数组：支持逗号分隔的多场景名
- 场景参考图：场景的视觉参考图片

**使用方式**：
- 场景卡 → 连线 → 提示词节点，提示词中可使用 `{{scene}}` 变量

### 3.5 物品卡节点

**用途**：定义关键道具/物品的设计约束。

**属性**：物品名、物品描述、物品参考图

**使用方式**：
- 物品卡 → 连线 → 提示词节点，提示词中可使用 `{{item}}` 变量

### 3.6 剧本节点

**用途**：输入剧本/场次文本，作为 AI 生成的上下文和分镜来源。

**属性**：
- 剧本文本：多行自由文本输入

**使用方式**：
- 脚本节点连续到提示词节点时，点击提示词的「刷新状态」按钮可将剧本内容合并入提示词

### 3.7 分镜节点

**用途**：定义单个镜头参数，支持镜头类型、对白、转场方式。

**属性**：
- 镜头序号、镜头描述、镜头类型（全景/中景/近景/特写等）、对白文本
- 绑定角色 ID、绑定场景 ID、绑定物品 ID

### 3.8 提示词节点

**用途**：提示词的中心枢纽——输入、编辑、优化提示词文本，整合所有上游信息。

**属性**：
- 提示词文本：自由文本输入，支持模板变量

**模板变量**：
| 变量 | 替换内容 | 来源节点 |
|------|----------|----------|
| `{{character}}` | 上游角色名 | 角色卡 |
| `{{characterAppearance}}` | 上游角色外观描述 | 角色卡 |
| `{{scene}}` | 上游场景名 | 场景卡 |
| `{{sceneDescription}}` | 上游场景描述 | 场景卡 |
| `{{item}}` | 上游物品名 | 物品卡 |
| `{{itemDescription}}` | 上游物品描述 | 物品卡 |
| `{{图片参考1}}` | 第一个上游图片文件路径 | 素材输入（图片） |
| `{{图片参考2}}` | 第二个上游图片文件路径 | 素材输入（图片） |
| `{{视频参考1}}` | 第一个上游视频文件路径 | 素材输入（视频） |
| `{{视频参考2}}` | 第二个上游视频文件路径 | 素材输入（视频） |

> 注：变量支持带 `{{}}` 双括号和不带括号两种写法。不带括号的写法（如 `图片参考1`）在执行时也会被自动替换。

**按钮功能**：
- **刷新状态**：自动收集所有上游节点信息，逐条拼入提示词文本
- **AI 优化**：调用 LLM API 对当前提示词进行上下文感知优化，结果以预览卡片展示，可点「采用」替换或「关闭」放弃

**素材引用标签**：当有素材节点连线到提示词时，文本上方自动显示可点击的变量标签（🎨 `+图1` / 🎬 `+视1`），点击后在光标位置插入对应变量。

### 3.9 GPT 图像生成节点

**用途**：调用 GPT Image 2 API 生成图像。

**属性**：
- 纵横比：1:1、16:9、9:16、4:3、3:4、21:9 等 15 种选择
- 生成数量：1-4 张
- 质量等级：自动、低、中、高
- 参考图 URL：可选的参考图链接（逗号分隔多张）

### 3.10 Seedance 视频生成节点

**用途**：调用 Seedance 2.0 API 生成视频。

**属性**：
- 模型 ID：标准模型 / 快速模型
- 生成模式：文生视频 / 图生视频（首帧）/ 图生视频（首尾帧）/ 多模态 / 视频编辑 / 视频扩展
- 分辨率：480p / 720p / 1080p
- 纵横比：21:9 / 16:9 / 4:3 / 1:1 / 3:4 / 9:16
- 视频时长：秒数
- 生成音频：开/关
- 返回尾帧：开/关
- 联网搜索：开/关
- 服务等级：默认 / Flex

### 3.11 输出节点

**用途**：配置生成结果的保存方式。

**属性**（多选）：
- 保存到本地：将结果文件写入本地文件夹
- 上传到 OSS：将结果上传至阿里云 OSS
- 同步到飞书：将结果写入飞书同步队列（JSONL 文件）

## 第四章 工作流执行

### 4.1 连线规则

- 节点左侧圆形手柄为**入口**（target），右侧为**出口**（source）
- 连线方向必须为**从左到右**（source → target），构成有向无环图（DAG）
- 不允许环路，不允许从同一个出口连出两条线到同一个入口

### 4.2 拓扑排序执行引擎

当用户点击「全部运行」或单个节点的「▶ 运行」按钮时，系统执行以下流程：

```
步骤 1：收集图中所有节点和边，构建邻接表 + 入度计数器
步骤 2：BFS 广度优先拓扑排序，确定节点执行顺序
步骤 3：按拓扑序依次处理每个生成节点（GPT Image / Seedance / Banana）：
  a. 遍历上游节点，按类型提取数据：
     - 角色卡 → 角色名、外观描述、面部/体型提示词、负面提示词、一致性种子
     - 场景卡 → 场景名、场景描述
     - 物品卡 → 物品名、物品描述
     - 剧本/分镜 → 文本内容
     - 素材输入 → 图片/视频文件路径（按类型自动分类编号）
     - 提示词 → 执行模板变量替换后的完整提示词文本
  b. 构建执行上下文（ExecutionContext）
  c. 替换模板变量（{{character}} → 实际角色名等）
  d. 匹配生成器适配器（generatorRegistry.get(type)）
  e. 调用 AI API，传入构建好的提示词 + 图片参考列表
  f. 收集结果，写入三路保存通道
步骤 4：检查时间线是否有未生成片段，发出提醒
```

### 4.3 生成上下文自动构建

系统在运行时自动构建的执行上下文包含以下字段：

| 字段 | 说明 | 示例 |
|------|------|------|
| prompt | 完整提示词文本（模板变量已替换） | "角色「蜂医」：医疗兵装束\n场景「航天基地」：..." |
| imageRefs | 上游图片参考文件路径列表 | ["nbc://assets/ref.png?path=..."] |
| negativePrompt | 负面提示词（合并上游多个角色卡的 negativePrompt） | "畸形手指, 模糊, 低质量" |
| consistencySeed | 一致性种子值（取第一个上游角色卡的 seed） | 123456789 |
| batchCount | 批量生成数量 | 1 |
| genParams | 生成器适配器的专属参数 | { aspectRatio: "16:9", quality: "high" } |

### 4.4 节点运行状态指示

每个节点右上角显示运行状态图标：

| 图标 | 状态 | 说明 |
|------|------|------|
| ▶ | 待运行 | 初始状态 |
| 🔄 | 运行中 | 当前正在调用 AI API |
| ✅ | 成功 | 生成完成，结果落在输出节点中 |
| ❌ | 失败 | 生成失败，详情见失败日志面板 |

### 4.5 并发控制与错误处理

- 批量生成任务使用 `Promise.allSettled` 并发执行
- 每个任务附带 `AbortController`，支持用户取消
- 单个任务失败不阻塞其他任务
- 失败的任务自动记录到失败日志（useLogStore），含错误信息、时间戳
- 成功的结果通过 `commitResults` 进入三路保存协调器

## 第五章 节点组功能

### 5.1 创建节点组

**方式一：Ctrl + 点击**
1. 按住 `Ctrl` 键，依次点击多个节点
2. 画布左下角显示"已选 N 个节点"提示
3. 点击提示中的「建组」按钮，输入组名

**方式二：快捷键**
1. 框选或 Ctrl+点击选择多个节点
2. 按 `Ctrl + G`，自动创建节点组

**方式三：右键菜单**
1. 右键点击节点，选择「加入组选择」
2. 重复操作设置多个节点后，点击「建组」

### 5.2 节点组的视觉表现

创建成功后，组成员节点外围出现**虚线边框**，框的颜色从预设色轮中自动分配：
- 边框上方 header 显示：圆点 + 组名 + 节点数量
- **▶ 运行按钮** — 点击后仅执行该组内生成节点（按拓扑排序，忽略组外节点）
- **✕ 解散按钮** — 删除节点组（不影响节点本身）

### 5.3 组独立运行

节点组的设计目标是实现"子工作流"——复杂的项目中，不同段落使用不同的 AI 模型和参数。双击组标签或点击组边框上的运行按钮，系统会：

1. 取全图所有节点和边
2. 根据组成员的 nodeId 集合过滤
3. 对过滤结果进行拓扑排序
4. 仅执行属于该组成员的生成节点

### 5.4 组数据持久化

节点组信息随项目数据一起保存：
- **自动保存**：每次画布状态变更，groups 随 nodes/edges 通过 debouncedAutoSave 写入 localStorage
- **工作流导出**：.nbc.json 文件的 metadata 中包含 groups 数组
- **工作流导入**：从 .nbc.json 恢复时同步恢复所有节点组

## 第六章 时间线分镜系统

### 6.1 分镜行管理

时间线面板位于底部区域，核心操作单元为**分镜行**（TimelineRow）。每行代表一个镜头：

| 行字段 | 说明 |
|--------|------|
| 标题 | 分镜名称，如"开场全景" |
| 时长 | 镜头持续秒数 |
| 关联节点 | 绑定的提示词/剧本/分镜/生成节点 |
| 角色/场景/物品 ID | 镜头关联的设定 |
| 镜头类型 | 全景/中景/近景/特写等 |
| 转场方式 | 硬切/淡入淡出/叠化/擦除等 |
| 对白文本 | 该镜头的对白内容 |

**操作**：
- **添加行**：点 + 按钮新增空白分镜行
- **编辑行**：点击行展开属性编辑
- **排序**：拖拽行列表调整顺序
- **删除行**：点行右侧 ✕ 按钮
- **从剧本导入**：输入剧本文本（每行一个镜头），自动解析镜头类型、时长、转场、对白，批量创建分镜行

### 6.2 素材池绑定

每个分镜行有独立的**图片绑定**（最多 4 个方案）和**视频绑定**：

```
行 #1: 开场全景
  ├── 图片方案 #1 ✅ 当前激活（点选切换）
  ├── 图片方案 #2
  ├── 图片方案 #3
  ├── 图片方案 #4
  └── 视频素材 ✅
```

素材来源：
- **拖拽填充**：从素材库拖拽图片/视频到指定行的槽位
- **生成自动填充**：AI 生成完成后，结果通过 `commitResults` → `useTimelineStore.addLegacyClip` 自动入库

### 6.3 时间线轨道

时间线分为多个轨道层，沿时间轴水平排列：

| 轨道 | 颜色 | 内容 |
|------|------|------|
| 视频轨 | 紫色 | 视频/图片片段（从分镜行自动构建） |
| 对白轨 | 绿色 | 音频片段 |
| BGM 轨 | 金色 | 背景音乐 |
| 字幕轨 | 蓝色 | 对白文本 |

分镜行的顺序决定视频轨上片段的排列顺序，每行的时长决定片段的持续时间长度。对话文本自动生成字幕轨上的文本片段。

## 第七章 素材库管理

### 7.1 素材索引与扫描

素材库支持三种数据来源：

**本地文件夹**：
- 首次启动自动扫描 `默认本地路径`（可通过设置面板修改）
- 点击「扫描本地文件夹」按钮 → Electron 系统级文件夹选择器 → 自动扫描并索引支持的媒体文件
- 支持的格式：`.png` `.jpg` `.jpeg` `.gif` `.webp` `.mp4` `.webm` `.mov` `.avi`
- 自动识别全景图（文件名含 `pano` 或 `360`）

**阿里云 OSS**：
- 配置 AccessKey 后，点击「阿里云 OSS」按钮拉取远程素材列表
- 支持上传、删除、标签同步

**飞书云盘**：
- 配置 App ID/App Secret + 文件夹 Token 后，点击「飞书云盘」拉取

### 7.2 素材浏览器操作

| 操作 | 方式 |
|------|------|
| 搜索 | 顶部搜索框输入关键词 |
| 筛选 | 按分类标签（GPT Image / Seedance / Character / Scene 等）或按项目 |
| 切换视图 | 网格视图 / 列表视图 |
| 查看大图 | 双击素材卡片，进入全屏查看器 |
| 拖入画布 | 从素材库拖拽素材到画布空白区，创建素材输入节点 |
| 拖入时间线 | 拖拽素材到时间线行的素材槽位 |
| 编辑标签 | 悬停素材卡片，点铅笔图标 |
| 删除 | 悬停素材卡片，点垃圾桶图标（可选仅从库移除或放入回收站） |

### 7.3 素材数据持久化

素材列表通过 Zustand `persist` 中间件自动保存到 localStorage（key: `nbc_asset_store`），仅持久化 `assets` 数组和 `defaultLocalPath`。应用重启后素材列表自动恢复。

本地素材使用 `nbc://` 自定义协议 URL 格式引用，确保文件路径跨会话稳定。

### 7.4 视频缩略图生成

视频文件在素材浏览器中显示时，系统通过前端 `VideoThumbnail` 组件实时截取首帧（0.1 秒位置）生成 JPEG 缩略图：
- 截取后写入双层缓存：内存 `Map`（同会话最快） + `sessionStorage`（页面刷新后仍在）
- Canvas 输出限制为 320px 宽，JPEG 质量 0.55，单帧约 5-15KB

## 第八章 高级功能

### 8.1 自定义节点尺寸

**操作**：选中节点后，右下角出现拖拽手柄，拖拽可调整节点宽度和高度。

系统实时响应拖拽（`onResize` 而非 `onResizeEnd`），尺寸变化通过 `updateNodeData` 写入 Zustand，存储为 `nodeWidth` 和 `nodeHeight` 并自动持久化。

### 8.2 右键菜单

右键点击画布上的任意节点，弹出上下文菜单：
- **复制**：将当前节点复制到剪贴板
- **粘贴**：在右键位置粘贴剪贴板中的节点（含连线关系，自动生成新 ID）
- **加入组选择**：将当前节点加入待建节点组
- **删除**：删除当前节点及关联连线

### 8.3 AI 提示词优化

提示词节点内置 LLM 优化功能：
1. 在提示词文本框中输入原始提示词
2. 点「AI 优化」按钮
3. 系统调用 LLM API（`/api/promptOptimize`），将原始提示词送入大模型进行上下文感知优化
4. 优化结果以预览卡片展示在原文本下方
5. 用户可选择「采用」（替换原文本）或「关闭」（保留原文本）

### 8.4 上下文感知刷新

点击提示词节点的「刷新状态」按钮：
1. 遍历所有上游节点（通过边查找所有 source → target = 当前节点的连线）
2. 按节点类型提取信息：
   - 角色卡 → 角色名 + 外观描述
   - 场景卡 → 场景名 + 场景描述
   - 物品卡 → 物品名 + 物品描述
   - 剧本 → 剧本全文
   - 分镜 → 镜头类型 + 描述 + 对白
3. 将提取的信息按格式拼入提示词文本框
4. 如上流有素材节点，自动列出可用的 `{{图片参考1}}` `{{视频参考1}}` 变量

### 8.5 nbc:// 自定义协议

NBC 注册了 `nbc://` 自定义协议用于本地文件引用：

- 格式：`nbc://assets/{文件名}?path={文件绝对路径(URL编码)}`
- 通过 Electron `protocol.handle` 处理，从磁盘读取文件并以正确的 MIME 类型 + CORS 头返回
- 同时支持工作流深度链接：`nbc://import?url=xxx` / `nbc://template?id=xxx`

### 8.6 API 双通道代理

为保证安全，AI API 调用不从前端直接发出，而是通过代理中转：

- **Web 开发模式**（`npm run dev`）：Vite 代理 → `/api/grsai` → GrsAI、`/api/ark` → 火山方舟、`/api/comfyui` → localhost:8188
- **Electron 桌面模式**：`api:fetch` IPC → 主进程 `net.request` 代理 → 解决 CORS

前端代码通过 `src/api/client.ts` 统一判断当前环境，自动选择走 Vite 代理通道或 Electron IPC 通道。

### 8.7 三路保存协调器

生成结果通过 `saveManager.ts` 协调三路保存：

| 通道 | 存储位置 | 用途 |
|------|----------|------|
| 本地 | 文档\NBC素材 | 直接文件写入 |
| OSS | 阿里云 OSS staging 目录 | 远程存储与分发 |
| 飞书 | JSONL 队列文件 | 由 Agent 定时消费同步到飞书 |

## 第九章 项目管理

### 9.1 创建与管理项目

**创建项目**：
1. 点击左侧「项目」面板
2. 点「+ 新建项目」输入名称
3. 自动切换到新项目，画布为空

**切换项目**：
- 在项目列表中点击目标项目名，画布自动加载该项目的工作流

**重命名项目**：
- 右键项目名 → 选择「重命名」

**删除项目**：
- 右键项目名 → 选择「删除」（不能删除当前活跃项目）

### 9.2 项目自动保存

系统每 400ms（防抖）自动将画布状态写入 localStorage：

- `nbc_projects`：项目元信息数组（id, name, 时间戳, nodeCount）
- `nbc_project_{id}`：每个项目的 nodes + edges + groups
- `nbc_active_project`：当前活跃项目 ID
- `nbc_timeline_v2_{id}`：时间线数据
- `nbc_asset_store`：素材库数据

工具栏的圆点指示器反映保存状态：
- 🟢 绿色：已保存
- 🟠 橙色闪烁：有未保存的更改（dirty 状态）

额外支持可配置的**自动备份**功能：每隔 N 分钟自动导出 .nbc.json 到备份目录。

### 9.3 导出工作流

**操作**：点击工具栏「导出」按钮。

系统构建 `NbcFile` 对象（含 version / projectName / nodes / edges / metadata），导出为 `.nbc.json` 文件。文件的 metadata 部分同时包含时间线数据和灵感板数据，确保完整工作流可迁移。

在桌面模式（Electron）下，通过 `dialog.showSaveDialog` 选择保存路径。在 Web 模式下，通过浏览器下载。

### 9.4 导入工作流

**操作**：点击工具栏「导入」按钮。

选择 `.nbc.json` 文件 → 解析 JSON → 恢复 nodes/edges/groups → 如有 timeline 数据则写入 localStorage 并触发 `timeline-imported` 事件 → 如有 inspiration 数据则通过 `useInspirationStore.importProjectData` 恢复。

导出的工作流文件可在不同项目、不同设备之间迁移分享。导入时自动迁移时间线 ID（将旧项目 ID 映射到当前项目 ID）。

### 9.5 清空画布

点击工具栏「清空」按钮，确认后清除所有节点和连线（不可撤销）。此操作也会清空节点组。

## 附录 B：常见问题

**Q：如何添加新的 AI 模型支持？**
A：在 `src/engine/generatorRegistry.ts` 中注册新的适配器，实现 `GeneratorAdapter` 接口的 `execute` 和 `buildParams` 方法。

**Q：为什么执行节点时提示"无上游信息"？**
A：提示词节点的上游没有连接任何节点。请将角色卡/场景卡/素材节点等连线到提示词左侧入点。

**Q：素材节点标题栏的变量标签为什么在连线后才出现？**
A：变量标签是实时计算的——只有连入提示词节点后，系统才能确定该素材在提示词上游的索引位置并生成标签。

**Q：生成结果保存在哪里？**
A：取决于输出节点的配置——本地保存在"文档\NBC素材"目录，OSS 保存在配置的 bucket 中，飞书保存在 JSONL 队列文件中。

**Q：如何将工作流分享给其他人？**
A：导出为 .nbc.json 文件（包含所有节点、连线、节点组、时间线数据），对方通过"导入"功能加载即可。
