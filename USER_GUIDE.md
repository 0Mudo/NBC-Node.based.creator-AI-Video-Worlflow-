## 1. NBC 是什么

NBC（Node-Based Creator）是一个**节点式素材创作器**，让你像搭积木一样编排 AI 图像和视频生成工作流。

**核心思路**：把创作过程拆成一个个「节点」，用连线串起来，一键批量生成。

```
角色卡（蜂医）──┐
                 ├──→ 提示词 ──→ GPT Image 2 ──→ 输出
场景卡（巴别塔）──┘
```

**支持的 AI 能力**：
- **GPT Image 2**：AI 图像生成（文生图 / 图生图）
- **Seedance 2.0**：AI 视频生成（文生视频 / 图生视频）
- **ComfyUI**：本地工作流调用
- **AI 提示词优化**：用 LLM 自动优化你的提示词

## 2. 安装与首次启动

### 2.1 安装

**Windows 用户**：
1. 获取 `NBC Setup.exe` 安装包
2. 双击运行，按提示完成安装
3. 桌面会出现「NBC 创作器」快捷方式

**绿色版（免安装）**：
1. 获取 `NBC-Portable.exe`
2. 放到任意文件夹，双击即可运行

### 2.2 首次启动 — 配置向导

第一次打开 NBC，会弹出**配置向导**，引导你填写 API Key。整个过程只需 1 分钟。

```
┌─────────────────────────────────────┐
│  欢迎使用 NBC                        │
│  节点式素材创作器 — 用节点画布编排     │
│  AI 图像和视频生成工作流。             │
│                                      │
│  让我们先配置 API Key，只需 1 分钟。   │
│                                      │
│  [跳过配置]              [开始配置 →] │
└─────────────────────────────────────┘
```

**第 1 步：GPT Image 2**
- 输入你的 GrsAI API Key
- 从 [grsai.dakka.com.cn](https://grsai.dakka.com.cn) 获取

**第 2 步：Seedance 2.0**
- 输入你的火山方舟 API Key
- 从 [火山方舟控制台](https://console.volcengine.com/ark) 获取

**第 3 步：AI 提示词优化**
- 输入 LLM API Key（支持 DeepSeek、OpenAI 等）
- 留空则跳过此功能

> 💡 **提示**：你可以点击「跳过全部」稍后在设置面板中配置。但至少需要配置一个图像生成 API 才能使用核心功能。

NBC 的界面分为 **5 个区域**：

```
├────────┬─────────────────────────────────┬───────────────┤
│        │                                 │               │
│  左栏  │         画布区域                │    右栏       │
│ 264px  │       （ReactFlow）             │   288px       │
│        │                                 │               │
│ 项目   │   在这里拖拽、连线、编排节点     │  属性检查器   │
│ 工作流 │                                 │  （Inspector）│
│ 素材库 │                                 │               │
│        │                                 │               │
├────────┴─────────────────────────────────┴───────────────┤
```

### 左栏（3 个标签页）

| 标签 | 功能 |
|------|------|

| **素材库** | 浏览本地素材文件夹 |

### 右栏 — 属性检查器

选中画布上的任意节点，右栏会显示该节点的详细参数，可以在这里编辑。

### 底部面板（4 个标签页）

| 标签 | 功能 |
|------|------|
| **生成队列** | 查看当前生成任务的进度和状态 |

| **时间线** | 分镜坑位管理 + 素材拖拽填充 |

## 4. 创建你的第一个项目

### 4.1 新建项目

1. 点击左栏「项目」标签
2. 点击「+ 新建项目」按钮
3. 输入项目名称，例如「蜂医-巴别塔」
4. 点击确认

项目会自动保存到浏览器本地存储中。

### 4.2 搭建工作流

从左栏的**节点面板**拖拽节点到画布上：

1. 拖入一个 **角色卡** 节点
2. 拖入一个 **场景卡** 节点
3. 拖入一个 **提示词** 节点
4. 拖入一个 **GPT Image 2** 节点
5. 拖入一个 **输出** 节点

### 4.3 连线

从一个节点的**输出端口**（右侧圆点）拖到另一个节点的**输入端口**（左侧圆点）：

```
角色卡 ──→ 提示词 ──→ GPT Image 2 ──→ 输出
场景卡 ──↗
```

连线表示数据流向：角色卡和场景卡的信息会自动注入到提示词中。

### 4.4 配置节点

点击任意节点，右栏的**属性检查器**会显示该节点的参数：

**角色卡节点**：
- 从下拉菜单选择预设角色（蜂医、露娜、威龙等）
- 或手动输入角色名和外观描述

**场景卡节点**：
- 从下拉菜单选择预设场景（巴别塔、航天基地等）
- 或手动输入场景名和描述

**提示词节点**：
- 输入你想要的画面描述
- 可以用 `{{character}}` 和 `{{scene}}` 作为模板变量
- 运行时会自动替换为角色卡和场景卡的内容

**GPT Image 2 节点**：
- 选择生成模式（文生图 / 图生图）
- 设置图片比例（1:1 / 16:9 / 9:16 等）
- 设置质量（low / medium / high）

### 4.5 运行

点击 GPT Image 2 节点上的 **▶️ 按钮**，或点击画布顶部工具栏的 **「全部执行」** 按钮。

生成完成后，结果会自动出现在**输出节点**和**素材池**中。

## 5. 节点详解

NBC 有 **9 种节点类型**，分为 4 类：

### 输入类节点

#### 📁 素材输入（AssetInputNode）
- **用途**：从本地素材库选择图片/视频作为输入
- **参数**：素材路径、素材类型
- **输出**：素材文件路径

#### 👤 角色卡（CharacterCardNode）
- **用途**：定义角色信息，注入到下游提示词
- **参数**：
  - 角色名：从预设列表选择或手动输入
  - 外观描述：角色的外貌、服装、气质等
- **输出**：角色名 + 外观描述文本
- **预设角色**：蜂医、老太、露娜、疾风、威龙（三角洲行动）

#### 🏔️ 场景卡（SceneCardNode）
- **用途**：定义场景信息，注入到下游提示词
- **参数**：
  - 场景名：从预设列表选择或手动输入
  - 场景描述：环境、光线、氛围等
- **输出**：场景名 + 场景描述文本
- **预设场景**：巴别塔、潮汐监狱、航天基地等 6 个

#### 📝 剧本/分镜（ScriptNode）
- **用途**：编写剧本或分镜脚本
- **参数**：剧本文本（支持多行）
- **输出**：剧本文本
- **用途**：可以批量导入到时间线分镜坑位

### 处理类节点

#### ✏️ 提示词（PromptNode）
- **用途**：编写 AI 生成的提示词
- **参数**：
  - 提示词文本
  - 支持模板变量：`{{character}}`、`{{scene}}`
- **输出**：替换变量后的完整提示词
- **特色**：支持 AI 一键优化（见第 8 节）

### 生成类节点

#### 🎨 GPT Image 2（GPTImageNode）
- **用途**：AI 图像生成
- **参数**：
  - 生成模式：文生图 / 图生图
  - 图片比例：1:1、16:9、9:16、4:3、3:4
  - 质量：low / medium / high
  - 参考图 URL（图生图模式）
- **输出**：生成的图片 URL

#### 🎬 Seedance 2.0（SeedanceNode）
- **用途**：AI 视频生成
- **参数**：
  - 模型：seedance-1.0-lite / seedance-1.0 / seedance-2.0
  - 模式：文生视频 / 图生视频
  - 分辨率：720p / 1080p
  - 时长：5 / 10 秒
  - 推理模式：极速 / 均衡 / 高品质
  - 有声/无声
  - 尾帧参考图（图生视频模式）
  - 联网搜索增强
- **输出**：生成的视频 URL

#### ⚙️ ComfyUI（ComfyUINode）
- **用途**：调用本地 ComfyUI 工作流
- **参数**：工作流 JSON、输入参数
- **输出**：生成结果

### 输出类节点

#### 📤 输出（OutputNode）
- **用途**：接收上游生成结果，自动保存
- **参数**：
  - 保存位置：本地 / OSS / 飞书队列
  - 文件名模板
- **功能**：生成完成后自动触发保存流程

## 6. 连线与工作流

### 6.1 如何连线

1. 把鼠标移到节点**右侧的圆点**（输出端口）
2. 按住鼠标左键，拖到另一个节点**左侧的圆点**（输入端口）
3. 松开鼠标，连线完成

### 6.2 数据流向

连线代表数据的流向。上游节点的输出会作为下游节点的输入：

```
角色卡（输出：角色名+外观）──┐
                              ├──→ 提示词（自动注入变量）──→ GPT Image 2（生成图像）
场景卡（输出：场景名+描述）──┘
```

### 6.3 模板变量替换

提示词节点支持模板变量，运行时会自动替换：

```
你写的：{{character}} 站在 {{scene}} 中，电影级光影

运行后：蜂医 站在 巴别塔废墟 中，电影级光影
```

### 6.4 删除连线

点击连线选中它，然后按 `Delete` 键或右键选择「删除」。

## 7. 运行工作流

### 7.1 单节点运行

点击生成类节点（GPT Image 2 / Seedance / ComfyUI）上的 **▶️ 按钮**，只运行这一个节点。

节点会自动收集所有上游节点的输出作为输入。

### 7.2 全部执行

点击画布顶部工具栏的 **「全部执行」** 按钮，按拓扑排序依次运行所有生成节点。

### 7.3 执行状态

每个节点右上角会显示状态图标：

| 图标 | 含义 |
|------|------|
| ▶️ | 待执行 |
| 🔄 | 执行中 |
| ✅ | 成功 |
| ❌ | 失败 |

### 7.4 生成队列

底部面板的「生成队列」标签页会显示所有任务的实时进度：

- 任务名称
- 当前状态（等待/运行中/成功/失败）
- 进度百分比
- 开始/完成时间

### 7.5 失败日志

如果生成失败，底部面板的「失败日志」标签页会记录：
- 失败的节点类型
- 错误信息
- 请求参数快照
- 时间戳

点击「详情」可以查看完整的错误报告。

## 8. AI 提示词优化

### 8.1 什么是提示词优化

NBC 内置了 AI 提示词优化功能。你只需要写一句简单的描述，AI 会自动帮你扩展成高质量的英文提示词，包含光线、构图、摄影术语等细节。

### 8.2 如何使用

1. 选中一个**提示词节点**
2. 在右栏的属性检查器中，输入你的提示词（中文或英文均可）
3. 点击 **「✨ AI 优化提示词」** 按钮
4. 等待几秒，优化结果会显示在下方

### 8.3 上下文感知

如果你的提示词节点连接了上游的**角色卡**和**场景卡**，AI 优化时会自动读取这些信息：

```
你的输入：一个战士站在废墟里

AI 读取的上下文：
- 角色卡 = 威龙（战术装备/面罩/冷峻气质）
- 场景卡 = 巴别塔（高塔废墟/黄昏/沙尘）

AI 输出：
cinematic wide shot, a stoic soldier in tactical gear and mask 
standing amid the ruins of Babel Tower at golden hour, dust 
particles floating in harsh diagonal light, shallow depth of 
field, photorealistic, 8K, ARRI Alexa
```

### 8.4 采用 / 再试 / 放弃

优化结果出现后，你有 3 个选择：

| 按钮 | 效果 |
|------|------|
| **✅ 采用** | 用优化结果替换当前提示词 |
| **🔄 再试** | 重新生成一个优化结果 |
| **❌ 放弃** | 丢弃优化结果，保留原文 |

### 8.5 前提条件

使用 AI 提示词优化需要在设置中配置 **LLM API Key**（支持 DeepSeek、OpenAI 等兼容接口）。

## 9. 时间线与分镜系统

### 9.1 什么是时间线

时间线是一个**分镜坑位管理系统**。你可以把视频项目拆成一个个镜头（坑位），然后用画布生成的素材去填充它们。

**核心比喻**：坑位 = 萝卜坑，素材 = 萝卜。把萝卜种到坑里，确保不漏镜头。

### 9.2 打开时间线

点击底部面板的 **「时间线」** 标签页。

### 9.3 创建坑位

有 3 种方式创建坑位：

**方式 1：手动添加**
1. 点击时间线工具栏的 **+** 按钮
2. 输入镜头描述，如「全景·巴别塔·25秒」
3. 输入时长（秒）
4. 点击「添加」

**方式 2：从文本导入**
1. 点击工具栏的 **▼** 按钮
2. 在文本框中每行写一个分镜描述：
   ```
   镜头1：蜂医从巴别塔高处跃下
   镜头2：疾风从侧面突入
   镜头3：威龙举枪瞄准
   ```
3. 点击「导入」

**方式 3：从剧本节点导入**
1. 先在画布上创建一个**剧本节点**，写好分镜
2. 点击时间线工具栏的 **📥** 按钮
3. 自动从剧本节点导入所有分镜

### 9.4 编辑坑位参数

点击坑位底部的「编辑参数」，可以设置：

| 参数 | 说明 |
|------|------|
| 镜头类型 | 全景 / 中景 / 近景 / 特写 / 远景 |
| 时长 | 1-30 秒 |
| 转场 | 硬切 / 淡入淡出 / 叠化 / 擦除 |

### 9.5 填充坑位

**素材池**位于时间线底部，显示所有通过画布生成的素材。

**拖拽填充**：
1. 从素材池中**按住**一个素材
2. **拖拽**到对应的坑位上
3. 松开鼠标，素材自动填入

**视觉反馈**：
- 🔴 红色边框 = 空坑位（未填充）
- 🟢 绿色边框 = 已填充
- 拖拽悬停时 = 高亮显示目标坑位

### 9.6 移除填充

点击坑位中素材右上角的 **✕** 按钮，可以移除填充，恢复为空坑位。

### 9.7 未填充提醒

- 时间线工具栏会显示未填充坑位数量
- 全部执行完成后，如果有空坑位，通知面板会发出警告
- 导出前可以检查：「还有 3 个坑位未填充」

## 10. 通知面板

### 10.1 什么是通知面板

通知面板是一个 **Agent 动态面板**，自动汇报工作流的运行状态。

### 10.2 通知类型

| 类型 | 图标 | 场景 |
|------|------|------|
| ✅ 成功 | 绿色勾 | 生成完成、保存成功 |
| ⚠️ 警告 | 黄色三角 | 有空坑位未填充 |
| ❌ 错误 | 红色叉 | 生成失败、API 错误 |
| ℹ️ 信息 | 蓝色圆 | 任务开始、导入完成 |

### 10.3 操作

- **点击通知** → 标记为已读
- **全部已读** → 一键标记所有通知为已读
- **清空通知** → 删除所有通知

### 10.4 未读计数

通知标签页会显示未读数量的红色角标。

## 11. 素材库

### 11.1 打开素材库

点击左栏的 **「素材库」** 标签页。

### 11.2 添加素材目录

1. 点击素材库顶部的 **「选择目录」** 按钮
2. 在弹出的对话框中选择一个包含图片/视频的文件夹
3. NBC 会自动扫描该文件夹中的所有媒体文件

### 11.3 支持的格式

- **图片**：PNG、JPG、JPEG、GIF、WebP
- **视频**：MP4、WebM、MOV、AVI

### 11.4 使用素材

素材库中的文件可以：
- 拖拽到画布上的**素材输入节点**
- 作为 GPT Image 2 图生图模式的参考图
- 作为 Seedance 图生视频模式的首帧图

## 12. 项目管理

### 12.1 创建项目

点击左栏「项目」标签 → 「+ 新建项目」→ 输入名称 → 确认

### 12.2 切换项目

在项目列表中点击任意项目，画布会自动切换到该项目的工作流。

### 12.3 重命名项目

点击项目名称旁边的 ✏️ 编辑图标，输入新名称。

### 12.4 删除项目

点击项目名称旁边的 🗑️ 删除图标，确认后删除。

> ⚠️ 删除项目会同时删除该项目的所有节点和连线数据，且无法恢复。

### 12.5 自动保存

NBC 会自动保存你的工作。每次修改节点、连线或参数，都会自动同步到本地存储。

## 15. API 设置

### 15.1 打开设置

点击顶部工具栏的 **⚙️ 设置** 按钮（齿轮图标）。

### 15.2 配置 API Provider

NBC 支持 4 个 API Provider：

| Provider | 用途 | 需要配置 |
|----------|------|----------|
| GPT Image 2 | AI 图像生成 | GrsAI API Key |
| Seedance 2.0 | AI 视频生成 | 火山方舟 API Key |
| ComfyUI | 本地工作流 | 本地地址（默认 localhost:8188） |
| LLM | AI 提示词优化 | API 端点 + API Key |

### 15.3 添加 API Key

1. 在设置面板中找到对应的 Provider
2. 点击 **「添加端点」**
3. 填写端点名称、URL、API Key
4. 点击保存

### 15.4 添加自定义 Provider

点击 **「+ 添加 Provider」**，填写：
- 名称
- 能力类型（文本生成图像 / 图像生成视频 / 工作流 / LLM）
- 图标和颜色
- 参数列表
- 端点列表

## 16. 团队协作流程

### 16.1 三人 MVP 配置

**角色分工**：
- **冬**（导演）：创建模板工作流、审核最终产出
- **A**（提示词师）：撰写提示词、跑 GPT Image 生成
- **B**（视频师）：跑 Seedance 视频生成、素材入库

### 16.2 协作流程

```
第 1 步：冬创建模板
├── 创建项目，搭建角色卡 + 场景卡 + 提示词（留空）+ 生成节点
├── 导出为 .nbc.json 文件
└── 上传到 OSS，飞书群发链接

第 2 步：A 填词生成图像
├── 点击飞书链接，NBC 自动导入工作流
├── 在提示词节点中填入描述
├── 运行 GPT Image 2 节点
├── 导出新版本：蜂医-巴别塔-A填词.nbc.json
└── 上传到 OSS，飞书群发新链接

第 3 步：B 生成视频
├── 点击飞书链接，NBC 自动导入 A 的版本
├── 接入 Seedance 节点
├── 运行视频生成
├── 素材自动入 OSS
└── 上传最终版

第 4 步：冬审核
├── 下载最终版 .nbc.json
├── 检查时间线坑位是否全部填充
└── 审核最终产出
```

### 16.3 文件命名规范

```
{日期}-{角色}-{场景}-{环节}.nbc.json

示例：
2026-05-09-蜂医-巴别塔-冬模板.nbc.json
2026-05-09-蜂医-巴别塔-A填词.nbc.json
2026-05-09-蜂医-巴别塔-B视频.nbc.json
```

### 16.4 OSS 目录结构

```
team-project/
├── templates/          # 冬维护的基准工作流
├── wip/                # 进行中的工作流
│   ├── 2026-05-09-蜂医-巴别塔-A填词.nbc.json
│   └── 2026-05-09-蜂医-巴别塔-B视频.nbc.json
├── final/              # 最终版
│   └── 2026-05-09-蜂医-巴别塔-完成.nbc.json
└── assets/             # 生成的素材
    ├── images/
    └── videos/
```

在飞书群中发送深度链接，团队成员点击即可一键导入：

## 18. 常见问题

### Q: 点击运行后没有反应？

**A**: 检查以下几点：
1. 是否已配置 API Key（设置面板）
2. 节点是否正确连线（生成节点需要有输入）
3. 检查底部「失败日志」是否有错误信息

### Q: 生成的图片/视频在哪里？

**A**: 
- 生成结果会显示在**输出节点**中
- 同时自动保存到本地（默认路径：`文档\NBC素材\`）
- 也会出现在底部**时间线**的素材池中

### Q: 提示词优化按钮是灰色的？

**A**: 
1. 检查是否配置了 LLM API Key（设置面板 → LLM）
2. 确保提示词文本框中有内容
3. 确保网络连接正常

### Q: 时间线的坑位怎么填充？

**A**: 
1. 先在画布上运行生成节点，产出素材
2. 素材会自动出现在时间线底部的**素材池**中
3. 从素材池**拖拽**素材到对应的坑位

### Q: 如何把工作流分享给队友？

**A**: 
1. 点击左栏「工作流」→「导出」，保存为 `.nbc.json`
2. 把文件上传到 OSS
3. 在飞书群发送深度链接：`nbc://import?url=OSS链接`
4. 队友点击链接，NBC 自动导入

### Q: 数据会丢失吗？

**A**: 
- 项目数据保存在浏览器本地存储中，关闭应用不会丢失
- 建议定期导出 `.nbc.json` 文件作为备份
- 清除浏览器数据会丢失所有项目

### Q: 支持多人同时编辑同一个工作流吗？

**A**: 
- 当前版本不支持实时协作
- 采用**接力模式**：每人导出新版本，通过 OSS + 飞书传递
- 未来版本会考虑加入实时协作功能

### Q: Electron 安全警告怎么消除？

**A**: 
- 开发模式下会显示 CSP 警告，这是正常的
- 打包后的正式版本不会显示此警告
